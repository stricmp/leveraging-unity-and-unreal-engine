DreadOut Development Challenges (Steam)

  • Our first title that made through Steam Greenlight
    • Also an Indonesian's first! :)
  • We have no idea about Steamworks
    • Only accessible once you get greenlit back then.
  • Steam DRM tool doesn't work with managed .NET
    • Only works on native C/C++ games.
  • So, native plugin need to be developed for Unity
    • Implemented in C & C++, glued to Unity with C#.
    • In retrospect, this opened the door to other market.

Part 4
Leveraging Unreal
(DreadEye)

DreadEye Development Challenges

  • Our first experience in using Unreal Engine 4
    • Unity experience help us learning UE4 faster.
  • UE4 already have Steamworks integrated
    • But we need features it didn't support.
  • We ended up creating our own integration
    • Integrated within the game code, not a plugin.
  • Already has many targets from the beginning
    • Oculus DK1/DK2/CV1, OSVR, VIVE/SteamVR.
    • VIVE/Viveport, FOVE VR, STOVE VR, Lenovo K5.

Unreal Engine 4 is our best fit engine for the project

  • UE4 full source code is available for free
    • Engine is C++. Build tool is C#.
    • Can be customized to suit our needs.
  • UE4 modular architecture ease up integration
    • Modifying independent module won't affect others.
    • Required modules can be specified at build time.
  • BuildTool enables further customization
    • PreBuild hook allows specific build requirement.
    • PostBuild hook allows specific package bundling.

Build customization: implementation

  • External dependencies were also included in project
    • Located outside of Source directory.
    • So Unreal will not 'see' it.
  • Simple 'lock file' configuration-switch mechanism
    • Builder creates a target-named file.
    • Then build the package as usual.
  • Build script decides how to build the package
    • It solves module and external library dependency.
    • Also defining specific code preprocessing flags.

Build Script: MyProject.Build.cs (1)

    public class MyProject : ModuleRules {
        public MyProject(ReadOnlyTargetRules Target) :
